================================================================
  PA Agent 智能体部署方法（Windows 版）
  适用环境：Windows + Python 3.11+ 已安装 + MT5 已安装
================================================================

【说明】
本文档供自动化 Agent 按步骤执行部署。每一步都有明确的命令和验收条件。
遇到问题时按"故障排查"章节处理。最后必须通过验收环节才能交付给用户。

【重要：先检测系统！】
Agent 在执行任何部署步骤之前，必须先检测当前操作系统：

  PowerShell 命令：
    [System.Environment]::OSVersion.Platform

  预期输出：Win32NT

  若输出不是 Win32NT（例如 Unix 或 Darwin），说明当前不是 Windows 系统：
  → 停止执行本文档，改用项目中的 "MAC版本智能体部署方法.txt"
  → Mac 版文档会指导 Agent 修改项目代码适配 macOS（默认数据源改为 TradingView 等）
  → Mac 版部署完成后，Agent 必须自动运行检测确认无错误

【重要：项目路径】
本文档中 PROJECT_ROOT 代表 PA_Agent 项目的根目录。
执行前必须先确定项目实际路径，后续所有命令中的 PROJECT_ROOT 都替换为实际路径。
例如：用户项目在 D:\cl\PA_Agent，则 PROJECT_ROOT = D:\cl\PA_Agent
      用户项目在 C:\Users\张三\Desktop\PA_Agent，则 PROJECT_ROOT = C:\Users\张三\Desktop\PA_Agent

确定方法：找到项目中存在 pa_agent\main.py 和 pyproject.toml 的那个目录，就是 PROJECT_ROOT。
Agent 执行时可用以下命令自动定位：
  $PROJECT_ROOT = (Get-ChildItem -Path C:\ -Recurse -Filter "pyproject.toml" -ErrorAction SilentlyContinue | Where-Object { Test-Path (Join-Path $_.DirectoryName "pa_agent\main.py") } | Select-Object -First 1).DirectoryName

【重要：不使用虚拟环境】
本方案直接将依赖安装到系统 Python，不创建虚拟环境。
原因：用户双击 bat 运行程序，虚拟环境会增加不必要的复杂度和闪退风险。
只要系统 Python 是专门给本程序用的，就不会有冲突问题。

【重要：Shell 说明】
本文档所有命令均使用 PowerShell 语法。不要使用 CMD 语法（如 cd /d、dir 等）。
若当前 shell 不是 PowerShell，请先切换：powershell

【故障排查】

问题 1：pip install 网络超时 / ConnectionError
解决：
  方案 A - 换清华源（已在上面命令中使用）：
    pip install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

  方案 B - 换阿里云源：
    pip install -e . -i https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com

  方案 C - 逐个安装（某个包卡住时单独装）：
    pip install PyQt6 -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
    pip install pyqtgraph -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
    ... 依次安装每个包

问题 2：PyQt6 安装后报 "DLL load failed"
解决：安装 Microsoft Visual C++ Redistributable
  下载地址：https://aka.ms/vs/17/release/vc_redist.x64.exe
  安装后重启电脑

问题 3：MetaTrader5 包安装失败
解决：
  - MetaTrader5 包仅支持 Windows，确认当前是 Windows 系统
  - 若 pip 安装报错，尝试指定版本：
    pip install MetaTrader5==5.0.45 -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn
  - 必须确保 MT5 终端已安装，否则 Python 包虽能装上但运行时会报错

问题 4：tvDatafeed 安装失败（git clone 超时 / 网络不可达）
说明：tvDatafeed 是 TradingView 数据源的核心依赖，
  从 GitHub 安装（git+https://github.com/rongardF/tvdatafeed.git），
  国内网络可能无法直接克隆。
解决：
  方案 A — 配置 git 代理（如果已有代理）：
    git config --global http.proxy http://127.0.0.1:7890
    git config --global https.proxy http://127.0.0.1:7890
    pip install git+https://github.com/rongardF/tvdatafeed.git

  方案 B — 手动下载 + 本地安装：
    浏览器打开 https://github.com/rongardF/tvdatafeed
    → Code → Download ZIP → 解压到本地
    → Set-Location 到解压目录 → pip install .

  方案 C — 如果仅需 MT5 数据源，可忽略此错误（不影响 MT5 功能）：
    注：TradingView 数据源将不可用，但不影响 MT5 数据源。

  方案 D — 使用 gitee 镜像（如存在）：
    搜索 gitee 上是否有 tvdatafeed 镜像仓库，克隆后用 pip install . 安装。

问题 4.5：tvDatafeed 安装后，TradingView 数据拉取全部报 "Connection to remote host was lost"
原因：tvDatafeed 2.1.0 存在一个已知 bug —— 其 WebSocket headers 参数传的是 JSON 字符串
  而非 Python dict，导致 websocket-client ≥1.8 版本拒绝握手（HTTP 400）。
解决：找到 tvDatafeed 安装目录下的 main.py，修改一行代码。

  定位文件：
    python -c "import tvDatafeed; import os; print(os.path.dirname(tvDatafeed.__file__))"
  输出类似：C:\Program Files\Python313\Lib\site-packages\tvDatafeed

  编辑该目录下的 main.py，找到第 35 行附近的：
    __ws_headers = json.dumps({"Origin": "https://data.tradingview.com"})
  改为：
    __ws_headers = {"Origin": "https://data.tradingview.com"}

  改完后验证（使用 daily interval，避免分钟线超时干扰验证结果）：
    python -c "from tvDatafeed import TvDatafeed, Interval; df = TvDatafeed().get_hist('XAUUSD', 'OANDA', Interval.in_daily, n_bars=2); print('OK:', len(df), 'bars')"
  预期输出：OK: 2 bars

  说明：此 bug 仅在 websocket-client ≥1.8 时触发。如果你用的是旧版 websocket-client，
  可能不受影响，但建议一并修复以兼容未来升级。

问题 4.6：tvDatafeed 安装后数据拉取变慢，切换品种/周期要等 40 秒甚至更久
原因：tvDatafeed 2.1.0 的 get_hist() 在每次调用时都会新建 WebSocket 连接，但从不关闭
  上一个连接——持续使用会积累大量半开放连接，触发 TradingView 速率限制。
  PA_Agent 项目已在 pa_agent/data/tradingview.py 中对此进行修复（无需改 tvDatafeed 源码）：
  1. 每次 get_hist() 调用完成后自动关闭 socket，不再泄漏连接
  2. 切换品种/周期时主动关闭当前 socket，立即中断挂起的请求（不再排队等超时）
  3. 单次请求超时从 15 秒缩短至 10 秒，且不再自动重试（避免超时翻倍）
  这些修复已包含在项目代码中，pip install -e . 后即生效，无需额外操作。

  注意：即使有上述修复，TradingView 匿名（nologin）模式下仍受 IP 速率限制。
  若经常拉取失败，最有效的解决方案是在 GUI 设置中填入 TradingView 账号用于登录。

问题 5：pywin32 安装后报 ImportError
解决：安装后需执行 post-install 脚本：
    python "$env:USERPROFILE\AppData\Local\Programs\Python\Python311\Scripts\pywin32_postinstall.py" -install
  注意：上面的路径要根据实际 Python 安装位置调整，可以用以下命令找到：
    python -c "import os, win32api; print(os.path.dirname(win32api.__file__))"
  然后到该目录的 Scripts 子目录下找 pywin32_postinstall.py

问题 6：pyqtgraph 与 PyQt6 版本不兼容
解决：指定兼容版本：
    pip install pyqtgraph>=0.13 -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

问题 7：numpy 版本冲突
解决：
    pip install numpy>=1.26 -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

================================================================
  第五步：生成运行智能体.bat 到项目根目录
================================================================

在 PROJECT_ROOT 目录下生成"运行智能体.bat"文件。
bat 文件必须放在项目根目录（与 run.py 同级），且使用 GBK 编码保存。

执行以下命令生成（PowerShell）：

  $batContent = @"
@echo off
chcp 936 >nul 2>&1
title PA Agent - AI K线分析助手
cd /d "%~dp0"
python run.py
if errorlevel 1 (
    echo.
    echo [错误] 程序异常退出，请查看上方错误信息。
    pause
)
"@
  [System.IO.File]::WriteAllText((Join-Path "PROJECT_ROOT" "运行智能体.bat"), $batContent, [System.Text.Encoding]::GetEncoding("gbk"))

说明：
- cd /d "%~dp0" 会自动切换到 bat 文件自身所在的目录，所以 bat 必须放在项目根目录
- 无论用户把项目放在哪个路径，只要 bat 在项目根目录就能正确运行
- 不需要激活虚拟环境，因为依赖直接装在系统 Python 中
- 使用 GBK 编码是因为 bat 文件含中文，cmd.exe 默认用 GBK 解释
- chcp 936 设置代码页为 GBK，与 GBK 编码的 bat 文件一致，避免中文乱码
- 注意：PowerShell here-string 的结束 "@ 必须顶格写（前面不能有空格）

验收：确认文件已生成且内容正确

  Test-Path "PROJECT_ROOT\运行智能体.bat"
  Get-Content "PROJECT_ROOT\运行智能体.bat" -Encoding Default

预期输出：
- Test-Path 返回 True
- Get-Content 显示上述 bat 文件完整内容

【故障排查】
- 若 bat 文件中文乱码：确认用 GBK 编码保存（上述命令已使用 GBK），且 chcp 设为 936
- 若双击闪退：bat 末尾已有 pause，异常退出时会暂停窗口显示错误信息
- 若 PowerShell here-string 缩进问题：确保结束的 "@ 顶格无多余空格，bat 内容不要有前导空格

================================================================
  附：TradingView 数据源使用说明
================================================================

1. 刷新间隔设置
   在 GUI 设置页面中可配置"刷新间隔(毫秒)"（refresh_interval_ms），默认 1000ms。
   该值在 TradingView 数据源下同样生效，但注意：TradingView nologin 模式
   对高频请求有限流，建议日线设 3000-5000ms，分钟线可设 1000-2000ms。
   PA_Agent 内置了防重叠调用和指数退避机制，连续失败时会自动延长等待时间。

2. "获取数据"按钮
   TradingView 数据源不会在启动时自动拉取数据。
   需手动点击工具栏中的"获取数据"按钮，之后会进入持续刷新模式。
   切换品种或周期后，RefreshLoop 会在下一个 tick 自动使用新的品种/周期拉取，
   无需再次点击"获取数据"按钮——切换会立即中断当前挂起的请求，
   而不是等上一个请求的超时周期结束。

3. 交易所选择
   下拉框中的交易所已按类别标注（如 SSE（A股）、HKEX（港股）、BINANCE 等），
   加密类交易所无后缀标注。
   不确定交易所时可选择"（自动）"，程序会自动探测。

4. 匿名模式限速说明
   未配置 TradingView 账号时使用匿名（nologin）模式，TradingView 会对匿名 IP
   施加请求限制，尤其是小周期（1m、5m）拉取更容易触发。
   若频繁出现超时，最有效的解决方案是在设置界面填入 TradingView 账号：
   设置 → TradingView → 用户名 / 密码（凭证加密存储，不会明文保存）。

================================================================
  附：常见运行时问题
================================================================

Q: 启动后 MT5 连接失败
A: 确保 MT5 终端已打开并已登录账户。MT5 必须在程序运行期间保持打开。

Q: TradingView 数据拉取超时 / 切换品种后长时间没有数据
A: 常见原因和处理顺序：
   1. 先确认 tvDatafeed headers bug 已修复（见第三步 问题 4.5），否则所有请求都会失败
   2. 检查网络能否访问 TradingView（国内直连可能需要代理）
   3. 小周期（1m/5m）更容易触发 TradingView 匿名限速，切换到 15m 以上测试
   4. 长期超时的根本解决方案：在设置界面配置 TradingView 账号登录

Q: 切换品种/周期后数据迟迟不更新
A: 项目已修复 tvDatafeed socket 泄漏问题（见 问题 4.6），切换操作会立即中断上一个
   挂起的请求。如果仍然卡顿，通常是网络问题而非程序问题，参考上一条排查。

Q: API 调用报网络错误
A: 检查网络连接；若使用代理需确认代理可用；确认 API Key 和 Base URL 配置正确。

Q: config/settings.json 损坏
A: 删除该文件，程序重启会自动生成默认配置：
   Remove-Item "PROJECT_ROOT\config\settings.json"

Q: 日志在哪里
A: PROJECT_ROOT\logs\pa_agent.log

Q: 如何更新程序
A:
   Set-Location PROJECT_ROOT
   git pull
   pip install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn

Q: 双击 bat 闪退
A: bat 文件已经包含异常暂停逻辑。如果仍然闪退，说明是 cmd 窗口本身没弹出来：
   - 右键 bat → 编辑，确认内容正确
   - 改用命令行运行 bat 观察输出：
     cmd /k "PROJECT_ROOT\运行智能体.bat"
